Introduction

Since I am now unable to keep it up, I have written this page in order for future webmasters to... eh... carry on the torch I guess. As you may or may not know, the website before this reconstruction wasn't too great. It wasn't uniformed and was a bit too simple. Getting more in-depth
SSI - If you don't know what SSI is, then you can skip this. SSI (server side includes) was not used for the simple reasons of: I wasn't sure if it was server-supported (although it probably is), and it might confuse future webmasters who didn't know what it was... as well as teachers/administrators who try to figure out what I did. I know if I used SSI it would make things much easier to change globally, but... sorry... I decided it would be a bad idea.

Tables - The layout, basically, was created (HTML-wise) using tables... a lot of them. If you don't understand tables and cannot work with them well, then please do not try to make a new layout. Please make sure you at least look at my coding, or understand how I did it before you try to do it yourself... if you don't know how to work with tables, you may end up messing something up and getting very frustrated.

Layers/CSS - Layers were not used due to browser compatibility. As least when I was at McNeil, a majority of the school was still using Netscape with a few using IE, which makes it very hard to use "advanced technology" such as CSS/layers and still have it render fine on both platforms. Very simple style sheets were used, just to make links be non-underlined:

                        <style><!--
                        A:link {text-decoration: none}
                        A:visited {text-decoration: none}
                        A:active {text-decoration: none}
                        A:hover {text-decoration: underline}
                        --></style>

WYSIWYG - Generally, I don't like WYSIWYG editors and prefer to do things "manually." But since many of the teachers/administrators use WYSIWYG editors, I used Frontpage 2000 to create a majority of the pages, to ensure that it was they would be able to edit some things on some pages alright. So as a result, even though you might like to do things "manually," try to at least feed your code through a WYSIWYG editor and make sure it comes out alright. This also influenced my non-SSI decision.

Directory Structure - Here is a visual, to help you know how the directory structure looks right after I graduated:

I tried to make a majority of the links relative instead of absolute (using a lot of ./, ../, ../../ instead of directly www.roundrockisd.org/mcneil/blablabla). For those of you who may not don't know:

  • ./ = stay in current folder

  • ../ = go one folder up

  • ../../ = go two folders up

  • ../../../ = I think you get the point =)

Images - When putting an image on a page, try to put a small black border around it like so: 

                  <img src="imageurl.jpg" border=1>

See the directory structure image as an example. The border makes the image/page look cleaner and more professional. There are of course, some images that just don't benefit from the border (such as non-rectangular/square images that have a transparent background, etc.)... just use your best judgment. Photos really benefit from the border, however.

Misc. - Please, please, please... use ONLY the font Arial... unless if you are doing a club page, or something else like that. Do not leave the font as the default Times New Roman, or something ugly like that. =) Also, try not to put too many colors and/or different font variations into one page. Try to keep it simple, professional, etc.

 
Now... getting to the "hard" part
 Since the main reason of this page is to help you create a new page, I might as well get to that now. First off, you'll probably need this file:
  • mcneil.zip (256KB)

And to help, here is a blank template page:

  • template.htm

Depending on where your file is in the directory/folder tree, you'll need a different number of "./" or "../" when linking to certain files/locations. I often find the "Replace" feature nice in those circumstances. =)

For the default layout, all the the image files needed are at root (we'll call root www.roundrockisd.org/mcneil/ since we'll never be touching www.roundrockisd.org). If you're making a new layout for one page, put all the files it needs into its own folder, don't dump everything into root... that makes it cluttered, messy and confusing. Basically the only thing I have organized is my website's directory structure... everything else, my room, desk, bed, shelves... is all a mess. =) Please try to keep it that way, it helps people in the future who are trying to figure out what you did.

Basically, what you will need to do is edit the file found in mcneil.zip (see next section for more details), then crop it up into many different pieces to throw into the tabled layout. The easiest thing to do, I think, would be to feed the template into a WYSIWYG editor, then replace images with the new images you've created. Keeping the same dimensions between your new cropped up images and the images used on this/the template page would be nice... although it can be time-consuming sometimes. Editing the image
 First you'll need to download the file mentioned above, mcneil.zip. You should find mcneil.psd when you unzip it. This is a Photoshop image file... I know a lot of you may not have Adobe Photoshop, so I suggest you use Jasc's Paint Shop Pro, the free (but with semi-steep learning curve) GIMP, or another image editor of equal/greater caliber that can handle layers fine.

Probably the only thing you will need/want to do is change the background color (should be the very bottom layer), the "Mavs!" text on the top-left image, and the top-right image... just play around with it I guess. 

The most time consuming part would probably be chopping up the image into all the different pieces... in which case... umm... good luck. =)
